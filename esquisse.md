エスキースの目的は，思考の過程をアウトプットし，アイデアを具現化することにある．

ユーザーは以下の２つどちらかを選択し，思考の連鎖を始める．AIが都度質問を作成し，ユーザーが答える，の繰り返し．

以下はスキームのイメージ．ボタンを押していき，質問に都度答えていく．メモリに内容は保存される．

- 順算：内発的動機から始める
  - あなたは以下のどちらですか？
    - 内発的動機に溢れている人 => 何を作るか，それにどんな意味があるか，誰の課題を解決するか，
    - 内発的動機の言語化をしたい人=> 内発的動機を探す => あなたが最も高揚した瞬間はなんですか？
- 逆算：イシューから始める
  - あなたは以下のどちらですか？
    - 解決したい課題がある => なぜ解決されていない？
    - 課題を探している　=> 今，世の中で必要とされている課題を列挙し，あなたがやるべきことを見つける

順算を選んだ場合，最終的にAIが，このプロジェクトがどのようなイシュー，つまり社会課題や技術的課題を解決するポテンシャルを秘めているか説明する．
逆算を選んだ場合，最終的に「なぜあなたがやるのか？」「あなたである必要があるのか？」「未踏（つまり政府）があなたに投資する必然性は何か？」という問いに収束する．

エスキースが完成したら，"資料に反映する"ボタンが使用可能になり，エスキースの内容をもとに編集ページの空欄が埋められる．この際，OPENROUTERを使用する．モデルは"google/gemma-3-27b-it:free"
